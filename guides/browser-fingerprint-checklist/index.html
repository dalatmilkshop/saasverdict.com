Browser Fingerprint Checklist (Pre-Scale Validation)

Before scaling account operations, validate each profile against a fixed checklist. This reduces drift, avoids fragile setups and simplifies debugging under load.

Profile Identity Layer

  • User agent matches chosen browser and OS profile.
  • Language and timezone align with proxy geography.
  • Screen size and device class are coherent.

Runtime Signals

  • Canvas/WebGL outputs are stable across re-opens.
  • Audio and font signals do not jump unexpectedly.
  • Navigator fields keep coherent values.

Network Consistency

  • IP geolocation matches timezone and language assumptions.
  • No unexpected DNS or route mismatch indicators.
  • Proxy quality is tested for latency and stability.

Operational Controls

  • Profile start and stop events are logged.
  • Retry policy is defined for API and navigation timeouts.
  • Failed profile runs are quarantined for review.

Recommended Validation Flow

  1. Create profile with deterministic settings.
  2. Start via API and capture runtime signals.
  3. Run 3 repeated sessions and compare consistency.
  4. Move only stable profiles into production queue.

Pair this checklist with the implementation guide for complete launch and cleanup routines.
